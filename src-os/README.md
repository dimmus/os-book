# src-os (phase 0 tutorial tree)

This is a standalone tutorial milestone:

- builds a minimal freestanding kernel (`HELLO INIT`)
- embeds/loads it from a standalone UEFI application
- calls `ExitBootServices`
- jumps to the kernel entry
- prints to COM1 (serial)
- runs under QEMU with OVMF

## Run

From `src-os/`:

```sh
make run
```

If your system OVMF path differs, set:

```sh
make run OVMF_FD=/path/to/OVMF.fd
```

## Expected output

On `make run`, QEMU serial should show:

```text

UEFI: alloc kernel pages
UEFI: alloc stack pages
```

## How `uefi/kernel_blob.inc` is produced

The UEFI app embeds the kernel as a generated C array.

Build pipeline:

1. `kernel/Makefile` builds `build/kernel.elf` from `kernel/init.cpp`.
2. `kernel/Makefile` converts ELF -> raw binary:
   - `llvm-objcopy -O binary build/kernel.elf build/kernel.raw`
3. `uefi/Makefile` runs:
   - `python3 gen_kernel_blob.py <kernel.raw> <kernel_blob.inc>`
4. `uefi/gen_kernel_blob.py` reads `build/kernel.raw` and writes `uefi/kernel_blob.inc`
   with:
   - `kernel_blob[]` (byte array)
   - `kernel_blob_len` (size in bytes)

So the source for `gen_kernel_blob.py` is always the raw file generated by the
kernel build step: `build/kernel.raw`.
